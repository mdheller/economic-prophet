# Human Economic Profit × World Economy Digital Twin Alignment v0.1

## Alignment thesis

Human Economic Profit is the value calculus. The World Economy Digital Twin is the bounded measurement, approval, custody, and control mesh that prevents the value calculus from becoming ungoverned machine authority.

The correct integration is not to let the twin optimize the economy directly. The correct integration is to let the twin observe, replay, score, gate, refuse, and seal economic-value states inside explicit corridors.

## Runtime boundary inherited from digital twin v2.9

The digital twin handbook states that the active runtime baseline remains v2.8, with an amber federation posture, five bounded corridors, pending approvals, auto-approved actions, integrity chain status, and seal verification visible. It also states that the external-integration behavior is simulated rather than production-grade.

Therefore Human Economic Profit must enter the twin as a governed proxy-hardening program, not as expanded machine authority.

## Proxy-hardening ladder

Upper-layer metrics must not become authority merely because they are computed. They graduate through explicit stages:

1. Named proxy.
2. Governed definition.
3. Source mapping.
4. Bounded replay.
5. Evidence custody.
6. Corridor approval integration.
7. Runtime hardening.
8. Promotion candidate.

EP_R, EP_Civ, FEQ, and narrative-state measurement should be treated as governed proxies until they have source maps, uncertainty posture, and sealed evidence.

## Authority-band alignment

| Band | Allowed HEP Use |
|---|---|
| A0 | advisory, replay, metric publication, inconsistency flag |
| A1 | small buffer adjustment request, verification request, repair task |
| A2 | demurrage bound change, alpha-share bound change, large civic offset approval |
| A3 | corridor policy change, issuance policy change, governance threshold change |
| A4 | cross-corridor executive action or hidden universal optimization — not allowed |

Under amber posture, A2/A3 actions must remain blocked or explicitly senior-gated. A4 is outside this framework.

## Evidence custody requirements

Every HEP runtime record should be compatible with the digital twin custody model:

- event records hash source evidence,
- EP records preserve term provenance,
- replay rows carry evidence grade,
- Heller balances reconcile from prior balances,
- civic assets carry used capacity and verification state,
- governance parameter changes produce signed decision events,
- blocked actions remain visible,
- seal receipts bind evidence to runtime state.

## Integration decision

Do not create a new Human Economic Profit corridor yet. Treat HEP as a cross-corridor semantic/economic layer consumed by the existing five bounded corridors. A new corridor should only be considered after schema hardening, evidence custody, replay maturity, and human approval surfaces are in place.

## Non-negotiable rule

Human Economic Profit must not widen Digital Twin authority. Digital Twin must not reduce Human Economic Profit to a financial proxy. The correct architecture is mutual discipline: HEP provides value ontology; the twin provides bounded runtime truth, approval, custody, visible refusal, and proxy hardening.
